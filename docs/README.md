# Dotfiles Docs

This directory keeps operational notes that are too detailed for the root README.

Read `docs/tasks.md` first when resuming work in this repo. It records the current focus, pending decisions, and validation notes.

Read `docs/fleet-sync.md` before planning any dotfile deployment to another machine.

Read `docs/ubuntu.md` when preparing a Linux host for these dotfiles.

Read `docs/node.md` when setting up Node, Bun, pnpm, or mise.
